Overview of DOM Events

DOM events are signals triggered by user actions or interactions on the webpage. Here, we explore common events such as onclick and onmouseenter.

Using the onclick Event

The onclick event fires when an element is clicked. You can attach a function to run whenever the event occurs. Note that only one function can be assigned per onclick handler.


// Select the first button element
let button = document.querySelector("button");

// Define the onclick event to display an alert when clicked
button.onclick = function () {
    alert("Button was clicked");
};

Assigning Functions to Events

Functions can be assigned directly to event handlers like onclick. Avoid calling the function directly (e.g., button2.onclick = btn();) as this will run the function immediately rather than on click.


// Select the button with id "btn"
let button2 = document.querySelector("#btn");

function btn() {
    alert("Hello there!");
}

button2.onclick = btn; // Assigns the function without executing it immediately

Using the onmouseenter Event

The onmouseenter event fires when the mouse enters the area of an element, such as hovering over a button. This is useful for adding effects or animations.


// Select the button with id "btn3"
let button3 = document.querySelector("#btn3");

// Define the onmouseenter event to log a message on hover
button3.onmouseenter = () => {
    console.log("You entered the area of button 3");
};

Why Use Event Listeners?

Direct event properties like onclick only support a single function per event. In contrast, event listeners allow multiple functions to respond to the same event on an element, offering flexibility and control over event handling.

Using addEventListener

The addEventListener method attaches a specified event and a callback function to execute when that event occurs. Its syntax is: Element.addEventListener(event, callback).


// Example with a single event listener on 'btn' button
let button2 = document.querySelector("#btn");

button2.addEventListener("click", () => {
   console.log("button clicked");
});

button2.addEventListener("click", () => {
  alert("button was clicked");
});
// Both functions execute upon clicking 'btn'.

Executing Multiple Actions on a Single Event

With addEventListener, multiple functions can respond to a single event, increasing interactivity. For instance:


// Two click events trigger both an alert and a console log
button2.addEventListener("click", () => {
   console.log("button clicked");
});

button2.addEventListener("click", () => {
  alert("button was clicked");
});

Other Event Types

Event listeners can handle various events, such as dblclick for double-click actions and mouseenter for hover effects.


// Double-click event on 'btn3'
let button3 = document.querySelector("#btn3");
button3.addEventListener("dblclick", () => {
   alert("button was clicked 2 times");  // Executes on double-click.
});

Event Listeners Beyond Buttons

Event listeners can be added to any HTML element, not just buttons. For example, hovering over a div can trigger a mouseenter or mouseleave event, allowing interactive effects like text changes.


// Adding event listeners to a 'div' element
let box = document.querySelector("div");

box.addEventListener("mouseenter", function(){
   box.innerText = "In Box";  // Text changes on hover.
});

box.addEventListener("mouseleave", function(){
   box.innerText = "";  // Text clears when mouse leaves.
});

Using 'this' in Event Listeners

The this keyword in an event listener callback refers to the element that the event listener is attached to. This can be useful when applying the same event listener to multiple elements.

Using 'this' with a Button


// 'this' in event listener callback refers to the button element
let btn = document.querySelector("#btn");
btn.addEventListener("click", function() {
    console.log(this); // Logs the button element
    console.dir(this); // Shows all properties of the button
    this.style.backgroundColor = "pink"; // Changes button color to pink
});

Applying the Same Event Listener to Multiple Elements

We can use this to handle multiple elements with a single function.


let h2 = document.querySelector("h2");
let h3 = document.querySelector("h3");
let para = document.querySelector("p");

function changeBackground() {
    console.log(this.innerText);
    console.dir(this);
    this.style.backgroundColor = "yellow"; // Changes background color to yellow
}

h2.addEventListener("click", changeBackground);
h3.addEventListener("click", changeBackground);
para.addEventListener("click", changeBackground);

Event Argument

When an event listener is triggered, an event argument is automatically generated and passed to the callback function, providing various details about the event.

Button Click Event


// Event argument in a button click
let btn = document.querySelector("#btn");
btn.addEventListener("click", function(event) {
    console.log("button was clicked");
    console.log(event);      // Logs the event object
    console.dir(event);      // Shows event properties
});

Keyboard Events

Keyboard events like keydown and keyup trigger when keys are pressed or released, respectively. These events are useful for interactive applications.

Basic Keyboard Event Handling


let input = document.querySelector("input");
input.addEventListener("keydown", function() {
    console.log("any key is pressed");
});

input.addEventListener("keyup", function() {
    console.log("any key is released");
});

Using Event Argument in Keyboard Events


let input2 = document.querySelector("#input2");
input2.addEventListener("keydown", function(event) {
    console.log(event);    // Logs event details including 'code' and 'key' properties
});

Accessing Key Code and Key Value

The event argument in keyboard events provides properties like code and key:


let input3 = document.querySelector("#input3");
input3.addEventListener("keydown", function(event) {
    console.log(event.code); // Logs the key code (e.g., "KeyF")
    console.log(event.key);  // Logs the actual key pressed (e.g., "f")
});

Tracking Arrow Key Movement


let input4 = document.querySelector("#input4");
input4.addEventListener("keydown", function(event) {
    console.log(`code = ${event.code}`);
    
    if (event.code == "ArrowUp") {
        console.log("character moves forward");
    } else if (event.code == "ArrowDown") {
        console.log("character moves backward");
    } else if (event.code == "ArrowLeft") {
        console.log("character moves left");
    } else if (event.code == "ArrowRight") {
        console.log("character moves right");
    }
});

Form Submission

Forms can be submitted programmatically, and specific actions (like page navigation) can be associated with their submission.

Form Submission with Alert


let form = document.querySelectorAll("form");

// Basic form submission
form[0].addEventListener("submit", function() {
    alert("Your form is submitted");
});

Preventing Default Form Action

To stop the form from executing its default action (such as navigating to a different page), you can use event.preventDefault().


form[1].addEventListener("submit", function(event) {
    event.preventDefault(); // Prevents the default action
    alert("Your form is submitted");
    console.log("Your form is submitted");
});

Explanation

  • event.preventDefault() is used to halt the form's default action, keeping the user on the same page.
  • This method is particularly useful when validating forms or sending data asynchronously.

Extracting Form Data on Submission

To capture data entered by a user, we can use JavaScript to extract values from form fields when the form is submitted. This is often achieved with the value property of input elements.

Basic Extraction Example


let form = document.querySelectorAll("form");

form[0].addEventListener("submit", function(event) {
    event.preventDefault(); // Prevents the form's default action
    alert("Your form is submitted");

    let input = document.querySelectorAll("input");
    console.log(input[0].value); // Displays the input value
});

Extracting Username and Password Values

This example shows how to capture multiple input values, such as a username and password, and store them in variables.


form[1].addEventListener("submit", function(event) {
    event.preventDefault();

    let user = document.querySelector("#user");
    let pass = document.querySelector("#pass");

    let username = user.value;
    let password = pass.value;

    console.log(`username = ${username}`);
    console.log(`password = ${password}`);
});

Using this.elements to Access Form Elements

You can access form elements without querySelector by using this.elements, which returns a collection of all elements within the form.


form[2].addEventListener("submit", function(event) {
    event.preventDefault();

    let user = this.elements[0];
    let pass = this.elements[1];

    let username = user.value;
    let password = pass.value;

    console.log(`username = ${username}`);
    console.log(`password = ${password}`);
});

Explanation

  • event.preventDefault(): Stops the form from navigating to a new page.
  • input.value: Accesses the user-entered value in an input field.
  • this.elements: A convenient way to access form inputs directly without additional selectors.

Understanding Change and Input Events

Change Event: Occurs when the value of an input, select, or textarea changes and the element loses focus.

Input Event: Fires whenever the value of an input, select, or textarea is modified, even by a single character.

Example: Change Event


let user = document.querySelector("#user");
let pass = document.querySelector("#pass");

user.addEventListener("change", function() {
    console.log("username changed");
    console.log("new value =", this.value);
});
pass.addEventListener("change", function() {
    console.log("password changed");
    console.log("new value =", this.value);
});

Example: Input Event

The input event will trigger whenever the user types each character.


let user2 = document.querySelector("#user2");
let pass2 = document.querySelector("#pass2");

user2.addEventListener("input", function() {
    console.log("username changed");
    console.log("new value =", this.value);
});
pass2.addEventListener("input", function() {
    console.log("password changed");
    console.log("new value =", this.value);
});

Dynamic Text Editor Example

This example demonstrates real-time display of input within a paragraph or dynamically created paragraphs below an editor.

Single Paragraph Update

let practice = document.querySelector("#practice");
let para = document.querySelector("p");

practice.addEventListener("input", function() {
    para.innerText = this.value;
});
Creating New Paragraphs Dynamically

let practice2 = document.querySelector("#newPractice");
let h2 = document.querySelector("h2");

practice2.addEventListener("input", function() {
    let newPara = document.createElement("p");
    newPara.innerHTML = `${this.value} <br>`;
    h2.append(newPara);
});

Explanation

  • Change Event: Triggers when the user exits the input field after modifying the value.
  • Input Event: Triggers on every character entered, deleted, or modified within an input field.
  • Dynamic Text Editor: Examples to display input in real-time in a paragraph or add new paragraphs for each update.

Understanding Event Bubbling

Event bubbling is a concept in which an event triggered on a child element propagates up to its parent elements. This means that when you click on a child element, the event first triggers the child’s event handler and then moves up to trigger the parent’s event handler.

Example of Event Bubbling


let div = document.querySelector("div");
let ul = document.querySelector("ul");
let li = document.querySelectorAll("li");

// Event handlers for bubbling
div.addEventListener("click", function() {
    console.log("div was clicked");
});

ul.addEventListener("click", function() {
    console.log("ul was clicked");
});

for (let i = 0; i < li.length; i++) {
    li[i].addEventListener("click", function() {
        console.log(`li ${i} was clicked`);
    });
}

Stopping Event Propagation

To prevent the event from bubbling up to parent elements, you can use the stopPropagation() method. This is useful when you want to handle an event on a child element without triggering the parent’s event handler.

Example: Stopping Propagation


// Stopping event bubbling
ul.addEventListener("click", function(event) {
    event.stopPropagation();
    console.log("ul was clicked");
});

for (let i = 0; i < li.length; i++) {
    li[i].addEventListener("click", function(event) {
        event.stopPropagation();
        console.log(`li ${i} was clicked`);
    });
}

Explanation

  • Event Bubbling: When a child element is clicked, its event triggers first, followed by the parent elements.
  • Stopping Propagation: Use event.stopPropagation() to prevent an event from bubbling up to parent elements, allowing for more controlled event handling.
