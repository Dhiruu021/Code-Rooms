What is CSS Grid?

CSS Grid is a two-dimensional layout system used to create complex grid-based designs. It allows you to define both rows and columns to form a grid, and place items within the grid. CSS Grid is often used in conjunction with Flexbox to create powerful, responsive layouts.

Grid Properties

To create a 2D grid with m rows and n columns, we use the following properties:

  • grid-template-rows: Defines the number and size of rows in the grid. Example: grid-template-rows: 1fr 1fr 1fr;
  • grid-template-columns: Defines the number and size of columns in the grid. Example: grid-template-columns: 1fr 2fr 1fr;
  • gap: Defines the spacing between rows and columns. Example: gap: 2rem; is shorthand for both row-gap and column-gap.
  • row-gap: Defines the gap between rows. Example: row-gap: 10px;
  • column-gap: Defines the gap between columns. Example: column-gap: 20px;

Fractional Units (fr)

The fr unit is used to distribute available space in a container. For example, if you set grid-template-columns: 1fr 1fr 1fr;, each column will take up 1/3 of the available space. If you use different values like grid-template-columns: 2fr 1fr 1fr;, the first column will take 2/4 (or 50%) of the space, and the other two will take 1/4 each (or 25%).

Declaring Grid Rows and Columns in Fractions

If you want to create multiple rows or columns with the same size, you can use the repeat() function:

grid-template-rows: repeat(10, 1fr);

This creates 10 rows, each with a size of 1 fraction of the available space.

Extending Grid Items

To extend the length or height of a grid item, you can use grid lines:

  • grid-column: Determines which column the item spans. Example: grid-column: 1 / 4; spans columns 1 to 3.
  • grid-row: Determines which row the item spans. Example: grid-row: 2 / 4; spans rows 2 to 3.

Shorthand Notations for Grid

You can use shorthand notations to combine the grid-column and grid-row properties:

  • grid-column: Combines grid-column-start and grid-column-end.
  • grid-row: Combines grid-row-start and grid-row-end.

Aligning Grid Items

  • justify-self: Aligns an item horizontally within its grid cell.
  • align-self: Aligns an item vertically within its grid cell.
  • place-self: A shorthand for both justify-self and align-self. Example: place-self: center; will center the item both horizontally and vertically.

Aligning Items in the Grid Container

  • justify-items: Aligns all grid items horizontally within the container.
  • align-items: Aligns all grid items vertically within the container.
  • place-items: A shorthand for both justify-items and align-items. Example: place-items: center; will center all items both horizontally and vertically within the grid container.

Conclusion

CSS Grid is a powerful tool for creating complex, two-dimensional layouts. By combining grid properties with fractional units, gap properties, and alignment features, you can create flexible and responsive layouts with ease.

What is the `repeat()` Function in CSS Grid?

The repeat() function is a shorthand used in CSS Grid to simplify the definition of grid-template-rows and grid-template-columns when you need to repeat certain values multiple times. This can save you from manually defining multiple identical values.

Key Features in This Example

  • Using `repeat()` for Rows: Instead of manually specifying each row size, you can use grid-template-rows: repeat(2, 1fr); to create two equal rows, each taking up 1 fraction of the available space.
  • Using `repeat()` for Columns: Similarly, grid-template-columns: repeat(3, 1fr); creates three equal columns, each occupying 1 fraction of the available space.
  • Flexible and Compact Layout: The repeat() function reduces repetition in your CSS code, making the layout more readable and maintainable.
  • Responsive Design: The use of fractional units (e.g., 1fr) ensures that the grid items adjust flexibly based on the available space in the container.

Understanding the Repeat Function

The repeat() function works by repeating a specific track size (either row or column) a given number of times. In this example:

  • grid-template-rows: repeat(2, 1fr); means there will be two rows, each taking up equal space (1 fraction each).
  • grid-template-columns: repeat(3, 1fr); means there will be three columns, each taking up equal space (1 fraction each).

Explanation of the Code

  • The * { margin: 0%; padding: 0%; box-sizing: border-box; } rule ensures uniform margin, padding, and box-sizing across all elements.
  • body takes up the entire viewport with width: 100vw; and height: 100vh;, ensuring the content fills the screen.
  • The .container class is the grid container, where grid-template-rows and grid-template-columns define the grid structure using the repeat() function.
  • row-gap and column-gap control the space between rows and columns, respectively, adding 2rem of space between each item.
  • The .item class represents individual grid items, each with a background color and a border for clarity.

Conclusion

This example demonstrates the use of the repeat() function in CSS Grid to define multiple rows and columns with equal size in a more compact and efficient way. The flexibility of CSS Grid allows for easily creating complex layouts that are both responsive and maintainable. The repeat() function is particularly useful when you need to repeat a pattern of rows or columns without redundancy in your code.

Understanding the Media Query

  • For larger screens (min-width: 326px): The grid layout switches to three columns with four rows using grid-template-columns: repeat(3, 1fr); and grid-template-rows: repeat(4, 1fr);.

  • Header and Footer: These elements span all three columns in larger screens using grid-column-start: 1; and grid-column-end: 4;.

Why Use Media Queries?

  • Responsive Design: Media queries allow the layout to adapt to different screen sizes, ensuring the content looks great on any device.
  • Better User Experience: By adjusting the layout for different screen sizes, users can navigate and view content easily, whether they are using a mobile phone, tablet, or desktop.

Additional Notes

Conclusion

This example shows how to use media queries to create a responsive CSS Grid layout that adapts to different screen sizes. Positioning Grid Items Using justify-self, align-self, and place-self

This example shows how to use CSS Grid's justify-self, align-self, and place-self properties to position grid items within the grid container.

Key Properties

  • justify-self: Controls the horizontal positioning of grid items. You can use values like end, start, center, stretch, and more.
  • align-self: Controls the vertical positioning of grid items. This works similarly to justify-self but for the vertical axis. Values include stretch, center, flex-start, flex-end, and baseline.
  • place-self: A shorthand property to set both justify-self and align-self at the same time.

Item Positioning Examples

  • justify-self: end; Positions the item to the end of the grid container along the horizontal axis.
  • justify-self: start; Positions the item at the start of the grid container along the horizontal axis.
  • justify-self: stretch; Stretches the grid item to take up the available space along the horizontal axis.
  • align-self: stretch; Stretches the grid item vertically to take up the available space.
  • place-self: start; A shorthand for setting both justify-self and align-self to start.

Understanding the Layout

  • Grid Items: Each item in the container has a fixed width and height of 100px, and uses justify-self to control horizontal placement.
  • Item Repositioning: The justify-self and align-self properties are used to reposition items along both axes. For example, justify-self: end; will move the item to the right, while align-self: stretch; will stretch the item vertically to fill the available space.

Shorthand for Item Positioning

The place-self property is a shorthand that combines both justify-self and align-self. For example, place-self: start; applies justify-self: start; and align-self: start; at the same time.

Conclusion

By using justify-self, align-self, and place-self, you can easily control the positioning of individual grid items within the container. These properties provide fine control over the layout, allowing for more flexible and responsive designs.

Positioning Grid Items Using justify-items, align-items, and place-items

This example shows how to use CSS Grid's justify-items, align-items, and place-items properties to position grid items within the grid container.

Key Properties

  • justify-items: Controls the horizontal positioning of grid items within their respective cells. You can use values like end, start, center, stretch, and more.
  • align-items: Controls the vertical positioning of grid items within their respective cells. Values include center, stretch, flex-start, flex-end, and baseline.
  • place-items: A shorthand property to set both justify-items and align-items at the same time. For example, place-items: flex-end; sets both properties to flex-end.

Item Positioning Examples

  • justify-items: last baseline; Positions items along the horizontal axis, aligning them to the last baseline of their cell.
  • justify-items: end; Aligns the items to the end of their cells along the horizontal axis.
  • align-items: center; Vertically centers items within their respective cells.
  • align-items: end; Aligns the items to the end of their cells along the vertical axis.
  • place-items: flex-end; A shorthand for justify-items: flex-end; and align-items: flex-end;, positioning items to the end of both axes.

Understanding the Layout

  • Grid Container: The container is a 3-column, 4-row grid, as defined by grid-template-columns: repeat(3, 1fr); and grid-template-rows: repeat(4, 1fr);. Items are placed in this grid structure with a gap of 2rem between them.
  • Item Positioning: The justify-items and align-items properties control the positioning of items horizontally and vertically within their respective grid cells. The place-items shorthand property can be used to set both properties simultaneously.

Conclusion

By using justify-items, align-items, and place-items, you can easily control the alignment and positioning of grid items within their cells in a grid layout. These properties allow for more precise control over the layout of elements and enable flexible, responsive designs.

Responsive Layout Using Grid (Without Media Queries)

Instead of using @media queries, CSS Grid can be used to create responsive layouts that automatically adjust the number of columns and rows based on the available space. Two main properties are involved in achieving this:

Key Properties

  • auto-fill: Automatically fills the grid with as many columns or rows as possible based on the available space.
  • minmax: Specifies the minimum and maximum size for a grid item. This allows the layout to adjust based on the viewport size, ensuring that the grid items scale appropriately.

Example Explained

In this example, we use grid-template-columns: repeat(auto-fit, minmax(200px, 500px));. Here's how it works:

  • auto-fit: The grid will automatically adjust the number of columns based on the available space. As the screen size changes, the number of columns will change accordingly.
  • minmax(200px, 500px): This ensures that the grid items will have a minimum width of 200px and a maximum width of 500px. If there's enough space, the items will grow to 500px, but they will shrink no smaller than 200px.

How It Responds to Different Screen Sizes

  • If the screen width is 2000px: 4 columns of 500px will be arranged horizontally.
  • If the screen width is 1000px: 2 columns of 500px will be arranged horizontally.
  • If the screen width is 1500px: 3 columns of 500px will be arranged horizontally.
  • If the screen width is 200px: Only 1 column of 200px will be displayed, as it is the minimum size defined.
  • If the screen width is 300px: 1 column of 200px will be shown.
  • If the screen width is 600px: 1 column of 500px will be displayed, and it will not shrink below 200px in width.

Advantages Over Media Queries

Using auto-fill and minmax for responsive layouts is simpler and less verbose compared to media queries. The grid automatically adjusts based on the available space, reducing the need for additional media query breakpoints.

Conclusion

CSS Grid's auto-fill and minmax properties provide a simple and effective way to create responsive layouts without relying on media queries. These properties ensure that your grid items automatically adjust in size and number based on the screen width, offering a fluid and adaptive layout for various screen sizes.

Grid Layout Using Grid Template Areas

Sample HTML Structure:

            
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
    <style>
        *{
            margin: 0%;
            padding: 0%;
            box-sizing: border-box;
        }

        html, body {
            height: 100%;
            width: 100%;
        }

        .container {
            display: grid;
            gap: 1rem;
            background-color: rgb(65, 59, 59);
            padding: 4rem;

            /* Instead of creating boxes in a grid format for each and need to expand rows and columns, we can just create a matrix using grid-area and it will automatically adjust the grid as per the matrix setup. */
            grid-template-areas: 
                "header header header"
                "sidebar content1 content1"  /* This creates a layout with 3 columns and 4 rows, with spanning rows and columns */
                "sidebar content2 content3"
                "footer footer footer";

        .items {
            background-color: rgb(112, 44, 56);
            border: 2px solid black;
            padding: 1rem;
        }

            grid-area: header;
        }

            grid-area: sidebar;
        }

        #content1 {
            grid-area: content1;
        }

        #content2 {
            grid-area: content2;
        }

        #content3 {
            grid-area: content3;
        }

        #footer {
            grid-area: footer;
        }
    </style>
</head>

Grid Layout Using Grid Template Areas

This layout utilizes the CSS Grid feature with the grid-template-areas property to create a simple and effective grid layout. By naming grid areas, you can define the structure of your layout easily and control how each element spans across the grid.

  • grid-template-areas: This property allows you to specify how grid items are placed within the grid by naming the areas. Each name corresponds to a section of the grid, and the layout will automatically adjust based on the definitions in the template.
  • grid-area: The grid-area property assigns each grid item to a specific area defined in grid-template-areas. For example, the element with id="header" will be placed in the "header" area.

Understanding the Layout

The grid is divided into a 3-column by 4-row matrix, with some items spanning multiple columns or rows:

  • header: The header spans across the entire top row, from the first to the third column.
  • sidebar: The sidebar spans two rows, occupying the first column.
  • content1: The content1 box spans across the second and third columns of the second row.
  • content2: Content2 takes the second row of the third column.
  • content3: Content3 occupies the third column on the third row.
  • footer: The footer spans the entire bottom row, from the first to the third column.

Why Use Grid Template Areas?

Using grid-template-areas allows for easier visual layout management. By defining named areas, you can quickly adjust and rearrange the layout without having to adjust the individual positioning of each item manually. This method simplifies the design process, especially for complex grid layouts.

Advantages

  • It is easy to understand and manage, as the grid layout is explicitly defined in a visual format.
  • It reduces the need for complex positioning and floating elements, simplifying CSS code.
  • It provides better control over responsive layouts, as you can rearrange grid items easily based on different screen sizes.

Visualizing the Grid

The grid layout defined by grid-template-areas visually represents a matrix with designated areas. This makes it intuitive to see how each element is positioned and how they span multiple columns or rows. For example, the header takes up the full top row, while the footer spans the bottom row.

Conclusion

Using grid-template-areas in CSS Grid is a powerful way to create flexible and easily understandable layouts. By naming grid areas and controlling how elements span, you can quickly adjust your design without complicating your code. This approach is particularly useful when working with complex layouts and responsive designs.
