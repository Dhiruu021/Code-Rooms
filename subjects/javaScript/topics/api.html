JavaScript APIs and Fetch Handling

API Overview

API (Application Programming Interface) is a way for applications to interact with external data or services, often through URLs that return data in JSON format.

JSON

JSON (JavaScript Object Notation) is a lightweight data-interchange format, similar to JavaScript objects but limited to valid JavaScript data types. Common in APIs, JSON data is typically not human-readable.

Example JSON Data


let jsonData = `{"fact": "Cats can rotate their ears 180 degrees.", "length": 45}`;
let parsedData = JSON.parse(jsonData);
console.log(parsedData.fact); // Access JSON data like an object

Convert Object to JSON

To create your own JSON data, convert a JavaScript object into a JSON string using JSON.stringify().


let student = { name: "Akhand", marks: 90 };
let jsonData = JSON.stringify(student);
console.log(jsonData);

Testing API Requests

  • Hoppscotch: A popular platform for testing APIs online.
  • Postman: Another option but requires signup and installation.

HTTP Verbs

HTTP verbs describe the type of operation requested:

  • GET: Retrieves data.
  • POST: Submits new data.
  • DELETE: Removes data.

HTTP Status Codes

  • 200: OK
  • 404: Not Found
  • 500: Internal Server Error

Fetching Data with Fetch

fetch() allows us to make API requests and returns a promise. For example, fetching a cat fact:


let url = "https://catfact.ninja/fact";
fetch(url)
    .then((response) => response.json())
    .then((data) => console.log(data.fact))
    .catch((error) => console.log("Error:", error));

Async/Await Example

Using async and await allows us to handle asynchronous code with a synchronous structure:


async function getCatFact() {
    try {
        let response = await fetch("https://catfact.ninja/fact");
        let data = await response.json();
        console.log(data.fact);
    } catch (error) {
        console.log("Error:", error);
    }
}
getCatFact();
